# Statistic Harness

Local-only, plugin-first statistical analysis harness.

## Quickstart

WSL/Windows friendly setup (includes dev deps):

PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
.\scripts\install_dev.ps1
stat-harness list-plugins
```

```bash
python -m venv .venv
. .venv/bin/activate
./scripts/install_dev.sh
stat-harness list-plugins
```

Or, from WSL, just run:

```bash
make dev
```

## Development

Run tests:

```bash
python -m pytest -q
```

## Feature Flags / Env

Default behavior is local-only. Optional features are guarded by env flags:

- `STAT_HARNESS_ENABLE_AUTH=1` enables UI/API auth (sessions + API keys).
- `STAT_HARNESS_ENABLE_TENANCY=1` enables tenant-aware isolation.
- `STAT_HARNESS_ENABLE_VECTOR_STORE=1` enables the sqlite-vec vector store.
- `STAT_HARNESS_SQLITE_VEC_PATH=/path/to/vec0.so` loads sqlite-vec if not builtin.
- `STAT_HARNESS_MAX_UPLOAD_BYTES` enforces upload size limits.

PII handling: `profile_basic` tags likely PII columns and stores hashed entities.
Any LLM/offsystem payloads generated by the prompt builder are anonymized using
the PII entity hash table.
